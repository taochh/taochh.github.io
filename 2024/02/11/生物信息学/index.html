1. 编译源代码

编译就是把高级语言变成计算机可以识别的 2 进制语言。可以分为编译型语言和解释型语言。

  • 编译型语言的是将源代码编译成二进制代码之后才能运行，因此执行效率更高，可已移植性更好。编译型语言代表有：C 语言，C++、Object-C 等；通常都会对源代码进行编译，生成可以执行的二进制代码，执行的是编译后的结果

  • 解释型语言的优点是不需要编译就可以直接运行，方便查看源代码。解释型语言代表有：JavaScript、Python、Erlang、PHP、Perl、Ruby 等；通常不用对源代码进行编译，一般是通过解释器载入脚本后运行。由于每个语句都是执行的时候才进行解释翻译，这样解释性语言每次执行就要翻译一次，效率相对要低。（Java 具有编译与解释两种特性）

软件编译过程

(1) configure

configure 是在编译前检查环境配置，也可以通过选项参数，来更改软件安装目录。configure是一个 shell 脚本文件，可以直接打开查看。configure 运行时会不停检查环境，提示一些warnings 和 error 信息。warnings 可以忽略，但是遇到 error 就会停止，需要解决这个依赖，然后重新运行 configure，直到全部检查通过，才可以进行下一步 make。

(2) make

当 configure 运行结束，且没有问题的时候，可以使用 make 进行编译。make 就是将源代码编译成二进制的过程。有些软件 make 之前还有一个 make test，make check 等过程。也有一些软件不需要 configure，直接 make 编译。make 结束之后就会在目录下发现有些可执行文件，或者多出一个 bin 目录。这个时候就可以直接运行这些软件了。

(3) make install

make 已经完成了编译过程，make install 主要是将软件链接到指定的安装目录，即第一步configue 指定的目录下。如果第一步没有指定，则安装到默认的目录下，一般是/usr 目录。这里需要注意，如果不是管理员用户，则没有权限写入/usr 目录，这个时候会提示一个权限的问题，“Permission denied”，这个不影响软件运行，可以手动将可执行程序链接至自己的软件目录内。这样 configure，make，make install 就完成了软件安装，这其中，最重要的就是 configure 这一步。如果 configure 没有问题，则 make 和 make install 一般可以顺利完成。

5.2 数据过滤

5.2.1 数据过滤原理

==需要过滤什么？==

  1. 非目片段的序列，如：Adapter接头序列、测序引物、barcode、index等

    • 为什么会测到 adapter? 空载（adapter之间无插入片段）、插入片段序列过短

  2. 含 N 碱基过多的序列

  3. 去除低质量序列，如小于 Q20

  4. 去除 duplication（按需）

关于 duplication

  • 两条 reads，reads1完全一致，reads2 完全一致。说明这两条reads来自基因组同一个位置，是由于打断不随机造成的 duplication
  • 拼接基因组时需要去除 duplication,否则会被认为是重复序列而去除这个片段
  • RNAseq 与 16S测序的 duplication 并不是打断不随机造成的，不能去除 duplication 序列，否则会造成丰度信息丢失

==数据过滤原则==

  • 不要求 100% 精确，原则是不影响后续分析
  • 可以根据最终结果，重新过滤数据

二、Pacbio

1. 测序原理

==优点==

  • 超长读长。PacBio RS:10-15kb，PacBio Sequel:5-70KB
  • SMRT的错误是随即发生的
  • 均匀的覆盖率：SMRT不需要扩增过程，不受GC偏向影响
  • 可直接检测碱基上的化学修饰
  • hifi reads 在准确性和长度之间达到平衡，99.999%

==缺点==

  • 数据量小，一张芯片只能测20G左右
  • 单分子测序原始数据错误率高，需要重复测序降低错误率
  • 价格较高，成本高于二代测序
  • 长度没有纳米孔测序长
  • 测序仪成本高，提高准确性需要牺牲长度和数据量。

==原理==

  • 单分子实时测序（SMRT）。基于光信号，边合成边测序
  • DNA聚合酶
  • ZMW 零模波导孔
  • 哑铃状文库

==碱基识别==

  1. 当聚合酶抓住一个dNTP的时候，会停留一段时间这时激发波长才会激发基团发出荧光，而孔中其他少量的游离dNTP则不会被激发
  2. 芯片的背面CMOS与测序Cell结合，每一个ZMW底部都对应一个CMOS感受器，可以将荧光信号转化为电信号从而完成碱基的识别与储存
  3. ZMW是一个直径只有10~50nm的孔，当激光打在ZMW底部时，只能照亮很小的区域，只有在这个区域内碱基携带的荧光基团被激活从而被检测到，大幅地降低了背景荧光干扰。
  4. 通过机器学习算法，即可将波信号转化成碱基序列

1.1 Pacbio测序结果

  1. Polymerase Read: 是一个测序聚合酶，在一个环状文库分子上，一直读取的序列，一般会很长，可能对环状分子读取了多遍。
  2. Subreads: Polymerase Read 的序列中，去掉哑铃状的测序接头序列，即可以得到多条subreads 序列。
  3. CLR (Standard Sequencing for Continuous Long Reads) ，称为超长测序模式，插入片段较长，产生的数据是基于单循环测序的结果。
  4. CCS(Circular Consensus Sequence): 环状一致性序列，是一个Polymerase read 上的多条subreads 序列，相互校正得到的一条反映真实文库的序列。
  5. Hifi reads: HiFi reads (High fidelity reads) 是Sequel ll三代测序平台推出的兼顾长读长和高准确度的测序序列，HiFi reads具有 >99.9% (Q20) 单分子reads准确度的准确率（Hifi reads一般 8-13kb）

基因组拼接原理

目前基因组拼接的算法主要有两种，一种是适用于长读长测序的基于有 overlap 的共有（consensus）序列组装的方法。这种算法被称为 Overlap Layout Consensus，简写为 OLC算法，另一种是用于短读长测序的基于 De Bruijn 图的短序列拼接

  • reads：直接测序出来的片段就称为 reads，不同测序平台测序出来的我们都可以称为 reads。

  • pairend reads：illumina 高通量测序的是双末端测序，reads 具有物理上的关系。小片段文库的这种物理关系叫做 pairend，大片段文库的这种关系称为 matepair 文库。

  • insertsize：illumina 双末端测序中间物理距离叫做 insertsize，其实就是随机打断后用来测序片段的长度。在序列拼接中会用到 reads 之间的这种关系，这个 insertsize 的值也非常重要。

  • overlap：reads 之间具有共有序列，这个共有序列关系成为 overlap；

  • kmer：将 reads 切割成固定长度的小片段。这样原来的 reads 就变成了很多的更短的 kmer。

    为什么要用 kmer？去除二代测序中的测序错误。包含测序错误碱基的 kmer 频数为 1，需要把这些 kmer 过滤掉。

  • contig：通过 kmer 之间的 overlap 连接成更长的片段，就是 contig。contig 是片段重叠群的意思。有时候我们也直接说 reads 通过 overlap 连接成 contig，都可以。准确来说是利用 kmer 连接成 contig；

  • scaffold：congtig 之间我们可以通过 reads 之间的 pairend 关系再连接成更长的片段，这个就叫做 scaffold，

  • gap：contig 连接 scaffold 过程中，中间区域使用 N 碱基来填充，称为 gap。

为了排除测序错误，需要先去除测序错误的位点，流程为切 kmer -> 统计相同 kmer 的频数 -> 删除 kmer 频数为1的 kmer -> 根据准确的 kmer 拼接成 contig

通过 kmer 估计基因组大小及杂合度

最左边的峰为测序错误峰；第二个峰为杂合峰，如果基因组杂合度比较低，则不会出现该峰；第三个峰为纯合峰，杂合峰的峰值所在位置一般为纯合峰所在位置的一半，基因组的杂合度越高，则杂合峰与纯合峰的比值越大；第四个峰为重复序列峰，在纯合峰值所在位置的两倍以上，重复序列越高，重复峰所占面积越大。教程

序列拼接步骤

Pregraph（构图）-> 生成 Contig -> 重新 Map -> 构建 Scaffold

  1. Pregraph（构图，De Bruijn 图）：切 kmer，根据 kmer 之间的 overlap 构图，将 kmer 连接起来（只有相差1个碱基的 kmer 才会被连接），最后形成 1 条长的 contig。contig 并不仅仅会出现 1 条，在最长的 contig 这条主干上会出现很多支流 contig，因为由于(1)测序错误存在（短的分支）、(2)多倍体杂合（形成bubble，随机选择1个SNP）、(3)存在重复序列（切除重复序列）、(4)去除低覆盖度分支

  2. mapping：将 reads 重新 map 到 contig 上，如果 reads1、reads2 落在同一个 contig 上，可以用于计算 insertsize，如果 reads1 在 contig1 上，reads2 在 contig2 上，则可以将 contig1、contig2 连接起来，连接成为 scaffold。

  3. scaffold：根据 mapping 的结果首先确定各个 contig 的先后关系，然后将其连接成为 scaffold，接着再对中间的空缺处用 N 填充（N的数量基于 insertsize，有时候并不是非常准确，因为 insertsize 的大小是不确定的）

    pairend太短？构建 matepair 文库、构建大片段文库、sanger测序补洞、三代测序补洞

软件在识别重复序列时会把 reads 数突然增多的判定为重复序列，因此基因组拼接不能有 duplication，否则会被软件误判为重复序列

基因组拼接结果评估

原则：评估准确性与完整性

  • N50：N50是基因组拼接之后一个评价指标。将拼接得到的所有的序列，根据序列大小从大到小进行排序，然后逐步开始累加，当加和长度超过总长一半时，加入的序列长度即为N50长度。N50越长，拼接得到的更长的序列越多。类似的还有N90等
